## Findings

### 1. The KA-NA root is real; the prefixes and suffixes are not noise

The family is too structured to dismiss:

- **base form**: `U-NA-KA-NA`
- **standard suffixed form**: `U-NA-KA-NA-SI`
- **prefix substitution**: `SE-KA-NA-SI`
- **infixed/extended form**: `U-NA-RU-KA-NA-SI`, `U-NA-RU-KA-NA-TI`
- **expanded prefix chain**: `JA-SA-U-NA-KA-NA-SI`

This is the strongest evidence that **KA-NA is a genuine root** inside a productive morphological family.

### 2. The ritual branch is much stronger than the administrative branch

`U-NA-KA-NA-SI` is robust in the integrated stack:

| Metric | Value |
|--------|-------|
| Frequency | 4 |
| Sites | IOZ, KOZ, PK |
| Threshold | STRONG |
| Final confidence | PROBABLE |
| Final assessment | `LUWIAN supported | Threshold: STRONG | Bayesian P=0.42 (BF=2.1) | Regional weight: 1.16` |

By contrast, standalone `KA-NA` is weak statistically:

| Metric | Value |
|--------|-------|
| Frequency | 2 |
| Sites | HT only |
| Threshold | ELIMINATED |
| Final confidence | POSSIBLE |

This means the research program should not overclaim the standalone admin form.

### 3. SYZa2 proves the ritual branch is tied to offering behavior

`U-NA-KA-NA-SI-OLE` on SYZa2 remains the single best interpretive control:

- the OLE determinative attaches directly to the verbal family
- that makes a broad "offering / provisioning / pouring" semantic zone much more plausible than an unrelated proper name

This is still not enough for a full dictionary-style translation, but it narrows the semantic field.

### 4. The best current bridge hypothesis is root-level, not full-form equivalence

The data support:

- **ritual branch**: offering/provision action family
- **admin branch**: provisioning / allocation / account label possibility

The data do **not** support:

- direct equation of standalone `KA-NA` with `U-NA-KA-NA-SI`
- promotion of standalone `KA-NA` to a translated lexeme

So the right claim is:

> `KA-NA` is probably a shared root operating across registers, with stronger ritual evidence than administrative evidence.

### 5. KHWc2005 keeps the family open inside KH, but not solved

`KA-NA-NI-TI` on `KHWc2005` is valuable because it shows the family is not confined to the classic libation corpus. But as a singleton, it expands the search space more than it resolves it.

## Working Interpretation

### Best supported claim

`KA-NA` is a **productive root family** whose ritual derivatives refer to an **offering/provision action**, while the two HT administrative occurrences may preserve the same root in a more account-oriented function.

### Confidence

**POSSIBLE** at the root level  
**PROBABLE** for `U-NA-KA-NA-SI` as the main ritual verb-family member

---

## What This Changes Operationally

1. **Do not reopen the language-family debate** on the basis of KA-NA alone.
2. **Do use KA-NA as a register-bridge clue** when comparing admin and ritual formulae.
3. **Do keep standalone `KA-NA` below promotion threshold** until new admin attestations appear.
4. **Do prioritize connected readings that can add a third administrative KA-NA-family occurrence**.

## Decision

**Outcome**: `strengthened hypothesis with HOLD`

The micro-study succeeded in narrowing the problem, but it did not create a promotion-grade breakthrough. The family is real, productive, and useful. The standalone administrative root remains under-evidenced.
